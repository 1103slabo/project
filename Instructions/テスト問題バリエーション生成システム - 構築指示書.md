# テスト問題バリエーション生成システム - 構築指示書

## 概要

テンプレートMDファイルに、複数のソースコードスニペットを差し込み、
Pandoc + Eisvogelテンプレートで複数バリエーションのPDFを一括生成する
Pythonスクリプトを作成してください。

## ディレクトリ構成

以下の構成でファイル・フォルダを作成してください。

```
project/
├── template.md          # 問題のテンプレート（プレースホルダー付き）
├── snippets/             # コードスニペット格納フォルダ
│   ├── q1_pattern_a.pseudo
│   ├── q1_pattern_b.pseudo
│   └── q1_pattern_c.pseudo
├── build.py              # ビルドスクリプト
└── output/               # PDF出力先フォルダ（自動作成）
```

## 1. template.md の作成

以下の内容で `template.md` を作成してください。
`{{CODE}}` の部分が、各スニペットファイルの内容に置換されます。

```markdown
---
title: "アルゴリズム問題 第1問"
---

# 問題

次のプログラムを読み、設問に答えよ。

\`\`\`pseudo
{{CODE}}
\`\`\`

## 設問1

このプログラムを実行した結果として正しいものを選べ。
```

## 2. snippets フォルダとサンプルファイル

`snippets/` フォルダを作成し、サンプルとして以下3つのファイルを作成してください
（内容は仮のサンプルコードでよい）。

- `snippets/q1_pattern_a.pseudo`
- `snippets/q1_pattern_b.pseudo`
- `snippets/q1_pattern_c.pseudo`

各ファイルには、簡単な擬似言語コード（例：変数宣言とループ処理など）を
適当に記述しておいてください。

## 3. build.py の作成

以下の要件を満たす `build.py` を作成してください。

### 要件

1. `output/` フォルダが存在しない場合は自動作成する
2. `variants` 辞書で「出力ファイル名（拡張子なし）」と「対応するスニペットファイルのパス」を対応付ける
3. `template.md` を読み込み、各バリエーションごとに `{{CODE}}` をスニペットの内容で置換する
4. 置換後の内容を一時MDファイル（`tmp_<variant名>.md`）として書き出す
5. Pandocを呼び出し、`--template eisvogel` を指定してPDFを生成する
   - 出力先: `output/<variant名>.pdf`
6. PDF生成後、一時MDファイルを削除する
7. 各バリエーションの処理結果（成功/失敗）をコンソールに表示する
8. Pandocの実行でエラーが発生した場合、そのエラー内容を表示しつつ
   他のバリエーションの処理は継続する（1つ失敗しても全体が止まらないようにする）

### 参考コード（このロジックをベースに、エラーハンドリング等を追加して実装してください）

```python
import subprocess
from pathlib import Path

variants = {
    "variant_a": "snippets/q1_pattern_a.pseudo",
    "variant_b": "snippets/q1_pattern_b.pseudo",
    "variant_c": "snippets/q1_pattern_c.pseudo",
}

Path("output").mkdir(exist_ok=True)

template = Path("template.md").read_text(encoding="utf-8")

for name, code_path in variants.items():
    code = Path(code_path).read_text(encoding="utf-8")
    content = template.replace("{{CODE}}", code)

    tmp_md = Path(f"tmp_{name}.md")
    tmp_md.write_text(content, encoding="utf-8")

    subprocess.run([
        "pandoc", str(tmp_md),
        "-o", f"output/{name}.pdf",
        "--template", "eisvogel",
    ])
    tmp_md.unlink()
```

## 4. README.md の作成（簡易）

プロジェクトルートに `README.md` を作成し、以下を簡潔に記載してください。

- このプロジェクトの概要（テスト問題のバリエーションPDFを一括生成する）
- 使い方: `python build.py` を実行すると `output/` フォルダにPDFが生成される
- バリエーションを追加する場合の手順:
  1. `snippets/` に新しいスニペットファイルを追加
  2. `build.py` の `variants` 辞書にエントリを追加

## 注意事項

- Pandocおよび `eisvogel.latex` テンプレートは、実行環境に別途インストール済みであることを前提とする（インストール処理は不要）
- 文字コードはすべてUTF-8で統一する
- コード内のコメントは日本語で記述してよい
