# README.md 修正指示書

## 概要

既存の `README.md` のタイトルおよびシステムの位置づけ説明を修正してください。
このシステムは特定の試験問題用ではなく、**汎用的なMarkdown差し込み印刷システム**
として位置づけを変更します。

## 修正内容

### 1. タイトルの変更

現在のタイトル：

```markdown
# FE疑似言語問題集 ビルドシステム
```

これを、以下に変更してください。

```markdown
# Markdown差し込み印刷システム
```

### 2. 「FE疑似言語問題」に関する記述の削除

README全体を確認し、「FE」「基本情報技術者試験」「疑似言語問題」など、
特定の試験・問題集に限定した表現が含まれている箇所をすべて削除、
または汎用的な表現に書き換えてください。

例：

- 「FE疑似言語問題のバリエーションを生成する」
  → 「テンプレートに任意のコード・画像・表などを差し込んで複数バリエーションの文書を生成する」
- 「擬似言語のコードスニペット」のような限定表現がある場合は、
  「コードスニペット」など汎用的な表現に置き換える

### 3. システムの位置づけを汎用システムとして明記

README冒頭（タイトル直下の概要部分）に、以下の趣旨の説明を追記・反映してください。

> このシステムは、特定の文書形式に限定されない汎用的な差し込み印刷システムです。
> テンプレートMDファイルに対して、コード・画像・表などの要素を差し込み、
> 複数バリエーションの文書（Markdown）を一括生成できます。
> 試験問題に限らず、どのような文書テンプレートにも応用可能です。

## 注意事項

- タイトル・概要文以外の構成（使い方、variantsの説明、プレースホルダー追加手順など）は
  変更不要です。内容自体は汎用的な書き方になっていればそのままで構いません。
- ファイル名やフォルダ構成（`snippets/`, `images/`, `tables/` など）の説明文中に
  試験問題を前提にした例文（変数名や問題文サンプルなど）が残っている場合も、
  汎用的な例（「サンプルコードA」など）に書き換えてください。
- README以外のファイル（template.md, build.py, snippets内のファイル等）は
  この指示書の対象外です。変更しないでください。
